Description

A complete, working NLP text classification pipeline built with Python, NLTK, and scikit-learn. The pipeline processes raw support case descriptions through five stages: tokenization and stop word removal, Porter stemming, TF-IDF feature extraction with bigrams, Multinomial Naive Bayes classification, and performance evaluation using precision, recall, and F1 metrics. The business problem it solves is automatic routing of Salesforce support cases to one of three queues: Access, Billing, or Technical. The artifact includes annotated Python source code, a visual pipeline diagram, a mapping of each technique to its Salesforce Einstein Language equivalent, and an evaluation of model strengths and limitations.

Objective

To design and implement an end-to-end NLP classification pipeline that demonstrates practical application of machine learning to text data, while connecting that implementation to enterprise deployment through Salesforce Einstein Language, showing that I can not only study AI concepts but build and apply them.

Process

  1. Defined the business problem: Salesforce support teams receive hundreds of unclassified cases daily. Manual triage is slow and inconsistent. An NLP classifier can route each case to the right queue in milliseconds, freeing agents to focus on resolution rather than sorting.
  2. Selected the tech stack based on industry standard tools: NLTK for text preprocessing primitives, scikit-learn for vectorization and classification, and Multinomial Naive Bayes as the baseline model, a proven first-choice for short-text classification tasks with small datasets.
  3. Built and annotated the preprocessing pipeline: lowercasing, punctuation removal, stop word filtering using NLTK's English corpus, and Porter stemming to normalize word variants (e.g., "billed," "billing," "bills" all reduce to "bill").
  4. Applied TF-IDF vectorization with unigrams and bigrams (max 500 features) to convert preprocessed text into numeric feature vectors. Bigrams capture phrases like "login fail" and "api error" that carry more signal than individual words.
  5. Trained and evaluated the Multinomial Naive Bayes model using an 80/20 train-test split, then measured performance using classification_report to validate precision, recall, and F1 score across all three categories.
  6. Mapped each pipeline step to its Salesforce Einstein Language equivalent, showing how this academic implementation scales into enterprise deployment through Salesforce's production NLP API.

Tools and Technologies

  • Python 3.10 - chosen because it is the dominant language in data science and ML, with the largest ecosystem of NLP libraries and the most employer demand for AI/ML roles.
  • NLTK 3.8 (Natural Language Toolkit) - used for tokenization, stop word filtering, and the Porter stemmer. NLTK provides well-documented, academically grounded NLP primitives widely taught in graduate programs.
  • scikit-learn 1.3 - used for TF-IDF vectorization, train-test splitting, Naive Bayes classification, and performance reporting. scikit-learn's consistent API and excellent documentation make it the industry baseline for traditional ML pipelines.
  • TF-IDF (Term Frequency-Inverse Document Frequency) - chosen over bag-of-words because it down-weights common terms that appear across all documents and up-weights terms that are distinctive to specific categories, producing better feature representations for classification.
  • Multinomial Naive Bayes - selected as the classifier because it performs strongly on short-text, sparse feature spaces (like support case descriptions), is interpretable, trains in milliseconds, and provides an excellent baseline before exploring more complex models.
  • Salesforce Einstein Language API - mapped to the pipeline as the enterprise deployment target, demonstrating how the Python prototype connects to production-grade NLP that can be deployed within Salesforce flows and automations.

How the NLP Classifier Works

Text does not come ready for machine learning. It must be transformed through a structured preprocessing pipeline before a model can learn from it. Each step below removes noise and increases signal.

1

Tokenization

Splits raw text into individual words and converts to lowercase. Removes punctuation so "login!" and "login" are treated identically.

2

Stop Word Removal

Removes common words like "the," "is," "and" that appear in every document and carry no classification signal. Reduces noise before vectorization.

3

Porter Stemming

Reduces words to their base form. "Billed," "billing," and "bills" all become "bill," so the model treats these variants as the same feature.

4

TF-IDF Vectorization

Converts text to numeric feature vectors. Words that appear frequently in one category but rarely in others receive higher weight, improving discrimination.

5

Naive Bayes Training

Learns the conditional probability that a set of features belongs to each class. Fast to train, interpretable, and highly effective on short text data.

6

Evaluation

Measures precision, recall, and F1 score per class. Identifies where the model misclassifies and guides retraining with better labeled examples.

nlp_case_classifier.py Python 
# NLP Case Classifier -- Intelligent Salesforce Support Routing
# Author: Deepthi Pinninti | IWU AI/ML Course, Workshop 5
# Stack:  Python 3.10 | scikit-learn 1.3 | NLTK 3.8

import re
import nltk
from nltk.corpus import stopwords
from nltk.stem import PorterStemmer
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.naive_bayes import MultinomialNB
from sklearn.model_selection import train_test_split
from sklearn.metrics import classification_report, accuracy_score

nltk.download('stopwords', quiet=True)

# ---- 1. TRAINING DATA: Salesforce Support Cases --------------------------------
cases = [
    "My login credentials are not working and I cannot access my account",
    "Account locked after multiple failed login attempts",
    "Password reset link is not arriving in my inbox",
    "Two-factor authentication code is expired before I can enter it",
    "I was charged twice for the same subscription this month",
    "My invoice shows an incorrect amount for the enterprise plan",
    "I canceled my subscription but was still billed this cycle",
    "Refund for overpayment has not appeared after 10 business days",
    "The REST API returns a 404 error on all POST requests",
    "Webhook events are not triggering on Opportunity record updates",
    "OAuth 2.0 token authentication is failing intermittently",
    "Integration with MuleSoft is dropping records during high volume",
]

labels = [
    "Access",    "Access",    "Access",    "Access",
    "Billing",   "Billing",   "Billing",   "Billing",
    "Technical", "Technical", "Technical", "Technical",
]

# ---- 2. TEXT PREPROCESSING -----------------------------------------------------
stemmer    = PorterStemmer()
stop_words = set(stopwords.words('english'))

def preprocess(text):
    text   = text.lower()
    text   = re.sub(r'[^a-z\s]', '', text)          # remove punctuation
    tokens = text.split()
    tokens = [t for t in tokens if t not in stop_words]  # drop stop words
    tokens = [stemmer.stem(t) for t in tokens]       # stem each token
    return ' '.join(tokens)

processed = [preprocess(c) for c in cases]

# ---- 3. TF-IDF FEATURE EXTRACTION ----------------------------------------------
# ngram_range=(1,2) captures both single words and two-word phrases
vectorizer = TfidfVectorizer(max_features=500, ngram_range=(1, 2))
X = vectorizer.fit_transform(processed)

# ---- 4. TRAIN / TEST SPLIT -----------------------------------------------------
X_train, X_test, y_train, y_test = train_test_split(
    X, labels, test_size=0.25, random_state=42, stratify=labels
)

# ---- 5. TRAIN NAIVE BAYES CLASSIFIER -------------------------------------------
model = MultinomialNB(alpha=0.1)   # alpha: Laplace smoothing
model.fit(X_train, y_train)

# ---- 6. EVALUATE MODEL PERFORMANCE ---------------------------------------------
y_pred = model.predict(X_test)
print(f"Accuracy: {accuracy_score(y_test, y_pred):.2f}\n")
print(classification_report(y_test, y_pred))

# ---- 7. CLASSIFY A NEW INCOMING CASE -------------------------------------------
new_case   = "My API integration keeps returning authentication errors on POST"
processed_new = preprocess(new_case)
vector        = vectorizer.transform([processed_new])
category      = model.predict(vector)[0]

print(f"New case: '{new_case}'")
print(f"Routed to: {category} Team")
# Output: Routed to: Technical Team
